VectorTileHub — Developer Guide

A reusable ASP.NET Core library for serving secure, cache-aware Mapbox Vector Tiles (MVT/PBF) from spatial databases — SQL Server, Oracle, or PostGIS.

Overview

VectorTileHub is a service library, not a framework. The single most important idea:

i

The library exposes no HTTP endpoints. It registers injectable services; your app owns the HTTP surface — you write the controllers, choose the routes, and apply your own authentication and authorization.

You consume three services and decide what to surface:

IVectorTileService

Produces an MVT/PBF tile for (layerId, z, x, y, variant).

IVectorTileLayerConfigProvider

Reads layer metadata (zoom range, extent, variants) and reloads config from disk.

IVectorTileCacheAdmin

Cache operations: generate, delete, invalidate, notify-change, swap, status.

✓

Convention: endpoints are MVC controllers ([ApiController] + ControllerBase), mapped via app.MapControllers() — never minimal APIs.

Install (from your local NuGet feed)

The packages are published to your local feed at F:\OneDrive\NUGetLocalPackages (version 1.0.0). A host needs two packages: the facade plus one provider.

1. Register the local feed

Add a nuget.config next to your solution (or edit the global one):

<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <packageSources>
    <add key="nuget.org" value="https://api.nuget.org/v3/index.json" />
    <add key="K1Soft.Local" value="F:\OneDrive\NUGetLocalPackages" />
  </packageSources>
</configuration>

2. Add the packages

dotnet add package K1Soft.IT.VectorTileHub.Server --version 1.0.0
dotnet add package K1Soft.IT.VectorTileHub.Providers.SqlServer --version 1.0.0
# or .Providers.Oracle / .Providers.Postgis

…which results in:

<ItemGroup>
  <PackageReference Include="K1Soft.IT.VectorTileHub.Server" Version="1.0.0" />
  <PackageReference Include="K1Soft.IT.VectorTileHub.Providers.SqlServer" Version="1.0.0" />
</ItemGroup>
i

The facade pulls in Core, Storage, Jobs, AspNetCore and Abstractions transitively — all from the same feed. Your host project should use the Web SDK (<Project Sdk="Microsoft.NET.Sdk.Web">).

✓

Re-publishing a new build? Bump <Version> in Directory.Build.props, then dotnet pack VectorTileHub.slnx -c Release -o "F:\OneDrive\NUGetLocalPackages". Clear the NuGet cache (dotnet nuget locals all --clear) if a consumer keeps resolving the old version.

Define a layer

Layers are JSON files discovered from LayerConfigFolder (relative to the content root). One file per layer, e.g. VectorTileHub/Layers/82-parcels.json.

{
  "id": 82,
  "layerKey": "parcels",
  "layerName": "Parcels Layer",
  "enabled": true,

  "provider": {
    "type": "SqlServer",                 // matches your AddVectorTileHub<Db>Provider()
    "connectionStringName": "Default",
    "tableName": "[GisDb].[dbo].[Parcels]",
    "idColumn": "Id",
    "geometryColumn": "Geom",
    "sourceSrid": 4326,                  // 4326 (lon/lat) or 3857 (Web Mercator)
    "commandTimeoutSeconds": 120         // null = 30s default, 0 = no timeout
  },

  "tile": {
    "minZoom": 12,
    "maxZoom": 17,
    "maxGenerationZoom": 17,             // caps cache GENERATION only (serving honors maxZoom)
    "extent": 4096,                      // MVT grid resolution within a tile
    "buffer": 64,
    "clipGeometry": true,
    "returnEmptyTileOutsideZoomRange": true,
    "allowOnDemandGeneration": true
  },

  "extent": { "minX": 38.93, "minY": 21.40, "maxX": 39.56, "maxY": 22.36, "srid": 4326 },

  "attributes": { "include": [ "PARCELNUMBER", "LAND_USES", "DISTRICT" ] },

  "cacheRules": [
    { "variantKey": "default", "isDefault": true },
    { "variantKey": "residential",
      "filter": { "column": "LAND_USES", "operator": "In", "values": [ "Villa", "Housing" ] } }
  ],

  "cache": { "enabled": true, "refreshPeriodMinutes": 1440 }
}

Full field docs in the Layer schema reference.

Register services (Program.cs)

using K1Soft.IT.VectorTileHub;                       // AddVectorTileHubServer
using K1Soft.IT.VectorTileHub.AspNetCore;            // UseVectorTileHubHangfireDashboard
using K1Soft.IT.VectorTileHub.Providers.SqlServer;   // AddVectorTileHubSqlServerProvider
using K1Soft.IT.VectorTileHub.Storage;               // EnsureVectorTileHubStorageAsync

var builder = WebApplication.CreateBuilder(args);

// 1) The host owns MVC. AddControllers() also feeds Swagger's API explorer.
builder.Services.AddControllers();

// 2) Your own auth — the library enforces none.
builder.Services.AddAuthentication(/* your scheme */);
builder.Services.AddAuthorization();

// 3) Register the VectorTileHub services (core + storage + jobs). Provider-agnostic.
builder.Services.AddVectorTileHubServer(builder.Configuration);

// 4) Register exactly the provider(s) your layers use.
builder.Services.AddVectorTileHubSqlServerProvider();
//     .AddVectorTileHubOracleProvider();
//     .AddVectorTileHubPostgisProvider();

var app = builder.Build();

// 5) Create/upgrade the internal SQLite settings store and warm its in-memory mirror.
await app.Services.EnsureVectorTileHubStorageAsync();

// 6) Host-owned pipeline — YOU decide what is exposed.
app.UseAuthentication();
app.UseAuthorization();

app.MapControllers();                                  // your tile/layer/admin controllers
app.MapHealthChecks("/vector-tile-hub/health");        // optional
app.UseVectorTileHubHangfireDashboard(/* auth filter */); // optional, opt-in, host-secured

app.Run();
  • AddVectorTileHubServer = core + storage + health-check registration + Hangfire jobs + IVectorTileCacheAdmin. Services only.
  • The provider is separate and pluggable — register one per data technology.
  • EnsureVectorTileHubStorageAsync() must run once at startup.

Your controllers

This is where you choose routes and security. Copy these from the Sample (TilesController, LayersController, CacheAdminController).

Tile endpoint

[ApiController]
[Route("vector-tile-hub")]
public sealed class TilesController : ControllerBase
{
    private readonly IVectorTileService _tiles;
    public TilesController(IVectorTileService tiles) => _tiles = tiles;

    [HttpGet("tiles/{layerId:int}/{z:int}/{x:int}/{y:int}.pbf")]
    public async Task<IActionResult> GetTile(int layerId, int z, int x, int y,
        [FromQuery] string? variant, CancellationToken ct)
    {
        // Host decides the variant: honor ?variant=, or derive from identity, e.g.
        //   variant = User.IsInRole("Resident") ? "residential" : null;
        var result = await _tiles.GetTileAsync(layerId, z, x, y, variant, ct);

        if (result.Status == VectorTileResultStatus.Ok)
        {
            Response.Headers["X-VTH-From-Cache"] = result.FromCache ? "true" : "false";
            Response.Headers["X-VTH-Stale"]      = result.IsStale   ? "true" : "false";
        }

        return result.Status switch
        {
            VectorTileResultStatus.Ok                 => File(result.TileBytes, result.ContentType),
            VectorTileResultStatus.NoContent          => NoContent(),                 // 204, outside extent
            VectorTileResultStatus.BadRequest         => BadRequest(new { error = result.Error }),
            VectorTileResultStatus.NotFound           => NotFound(new { error = result.Error }),
            VectorTileResultStatus.ServiceUnavailable => StatusCode(503, new { error = result.Error }),
            _                                         => StatusCode(500, new { error = result.Error })
        };
    }
}

Layer metadata endpoint

[ApiController]
[Route("vector-tile-hub")]
public sealed class LayersController : ControllerBase
{
    private readonly IVectorTileLayerConfigProvider _provider;
    public LayersController(IVectorTileLayerConfigProvider provider) => _provider = provider;

    [HttpGet("layers")]
    public IActionResult GetLayers() =>
        Ok(new { layers = _provider.GetAllLayers().Where(l => l.Enabled).Select(ToDto) });

    [HttpGet("layers/{layerId:int}")]
    public IActionResult GetLayer(int layerId)
    {
        var l = _provider.GetLayer(layerId);
        return l is null || !l.Enabled ? NotFound() : Ok(ToDto(l));
    }

    private static object ToDto(VectorTileLayerConfig l) => new
    {
        id = l.Id, layerName = l.LayerName,
        minZoom = l.Tile.MinZoom, maxZoom = l.Tile.MaxZoom,
        tileUrlTemplate = $"/vector-tile-hub/tiles/{l.Id}/{{z}}/{{x}}/{{y}}.pbf",
        variants = l.CacheRules.Count > 0
            ? l.CacheRules.Select(r => r.VariantKey).ToArray()
            : new[] { VectorTileVariant.DefaultKey },
        attributes = l.Attributes.Include
    };
}

Cache-admin endpoint (secured)

[ApiController]
[Route("vector-tile-hub/admin/layers")]
[Authorize(Roles = "GISAdmin")]                  // host-applied policy — the library enforces none
public sealed class CacheAdminController : ControllerBase
{
    private readonly IVectorTileCacheAdmin _admin;
    public CacheAdminController(IVectorTileCacheAdmin admin) => _admin = admin;

    [HttpPost("{layerId:int}/cache/generate")]
    public IActionResult Generate(int layerId, [FromBody] GenerateDto? r) =>
        Accepted(new { jobId = _admin.EnqueueGenerate(layerId, r?.MinZoom, r?.MaxZoom, r?.Variants, r?.MaxDegreeOfParallelism) });

    [HttpGet("{layerId:int}/cache/status")]
    public async Task<IActionResult> Status(int layerId, CancellationToken ct) =>
        await _admin.GetStatusAsync(layerId, ct) is { } s ? Ok(s) : NotFound();

    public sealed record GenerateDto(int? MinZoom, int? MaxZoom, string[]? Variants, int? MaxDegreeOfParallelism);
}
i

The host defines the HTTP request DTOs (the JSON shape is your concern). Service methods take primitive parameters — see the Service reference.

Security

  • Tiles — typically anonymous, or [Authorize]. To serve filtered data per user, map the role to a variant key and pass it to GetTileAsync.
  • Admin/config — guard with [Authorize(Roles = "…")] or a policy. These trigger destructive cache operations.
  • Hangfire dashboard — supply an IDashboardAuthorizationFilter; with none, Hangfire's default permits local requests only.
using Hangfire.Dashboard;

public sealed class DashboardAuthFilter : IDashboardAuthorizationFilter
{
    public bool Authorize(DashboardContext context) =>
        context.GetHttpContext().User?.IsInRole("GISAdmin") == true;
}

// Program.cs
app.UseVectorTileHubHangfireDashboard(new DashboardAuthFilter());

Database providers

Register the provider, then point each layer at it via provider.type (case-sensitive keyed name).

ProviderRegistrationprovider.typeDriver
SQL ServerAddVectorTileHubSqlServerProvider()"SqlServer"Microsoft.Data.SqlClient
OracleAddVectorTileHubOracleProvider()"Oracle"Oracle.ManagedDataAccess.Core
PostGISAddVectorTileHubPostgisProvider()"Postgis"Npgsql

You can register multiple providers; each layer picks one. All providers read the connection string from connectionString or connectionStringName, honor commandTimeoutSeconds, emit geometry as WKB, and intersect the tile envelope against the geometry column.

SRID handling

  • sourceSrid: 4326 — query envelope is converted to lon/lat for the spatial predicate; returned geometries are reprojected 4326 → 3857 for tiling.
  • sourceSrid: 3857 — used as-is. The stored geometry's SRID must match sourceSrid.

Spatial predicate per provider

ProviderWHERE (essence)
SQL ServerGeom.STIntersects(geometry::STGeomFromWKB(@envelope, @srid)) = 1
OracleSDO_RELATE(Geom, SDO_GEOMETRY(...), 'mask=ANYINTERACT') = 'TRUE'
PostGISST_Intersects(geom, ST_MakeEnvelope(@minx,@miny,@maxx,@maxy,@srid))
!

A spatial index is essential. Low-zoom tiles cover huge areas; without an index every tile triggers a table scan and queries time out. Create one bounded to your data extent and verify the optimizer uses it.

-- SQL Server (bound the grid to your data extent)
CREATE SPATIAL INDEX idx_Spatial_Parcels ON [dbo].[Parcels]([Geom])
USING GEOMETRY_AUTO_GRID
WITH (BOUNDING_BOX = (38.9, 21.4, 39.6, 22.4), CELLS_PER_OBJECT = 16, DROP_EXISTING = ON);

-- PostGIS
CREATE INDEX ON parcels USING GIST (geom);
i

PostGIS: variant filters cast the column to ::text because PostgreSQL won't implicitly compare a non-text column to a text parameter. Functionally identical to the other providers.

Caching & background jobs

Tiles are cached on disk (and optionally in memory). They enter the cache two ways:

  1. On-demand — a requested-but-uncached tile is generated, cached, and returned (when tile.allowOnDemandGeneration = true).
  2. Pre-generation — a Hangfire job renders a zoom range up front via IVectorTileCacheAdmin.EnqueueGenerate(...).
MethodEffect
EnqueueGenerate(layerId, minZoom?, maxZoom?, variants?, maxDegreeOfParallelism?)Background pre-generation. Returns the job id.
EnqueueDelete(layerId, cacheVersion?, deleteAllVersions)Background cache deletion.
EnqueueNotifyChange(layerId, minX, minY, maxX, maxY, srid, variants?)Background refresh of tiles in a bbox.
EnqueueSwap(layerId, newVersion?, regenerateAfterSwap, deleteOldVersion)Atomic blue/green version swap.
InvalidateAsync(...)Synchronous removal of cached tiles in a bbox. Returns null if the layer is unknown.
GetStatusAsync(layerId, ct)Active version, generation status, timestamps.

Performance knobs

  • maxDegreeOfParallelism — generation parallelism across tiles.
  • tile.maxGenerationZoom — caps generation depth; deeper zooms generate on demand.
  • layer.extent — bounds generation and builds an in-memory R-tree from a single full-layer query (far faster than one DB query per tile). Also enables extent gating: tile requests outside the extent short-circuit to 204 with no DB query.

Stale-while-revalidate

cache.refreshPeriodMinutes > 0 serves a cached tile immediately even when older than the window and enqueues a background refresh (X-VTH-Stale: true). 0 = never stale.

i

Hangfire uses in-memory storage by default — jobs don't survive a restart. Swap to durable Hangfire storage if you need persistence.

Variants (filtered caches)

A variant is an independently cached, separately addressable filtered view. The library is role-agnostic: the host maps a user/role to a variantKey and passes it to GetTileAsync; the variant's server-side filter scopes the rows.

"cacheRules": [
  { "variantKey": "default", "isDefault": true },
  { "variantKey": "residential",
    "filter": { "column": "LAND_USES", "operator": "In", "values": [ "Villa", "Housing" ] } }
]
OperatorPredicatevalues
Equalscol = @v0 (or IN if multiple)1+
Incol IN (@v0, @v1, …)1+
NotEqualscol <> @v01
IsNullcol IS NULL—
IsNotNullcol IS NOT NULL—

Values are always bound as parameters — never concatenated into SQL. Serve a variant with ?variant=residential; each variant has its own cache entries and its own slot in pre-generation.

Zoom ranges & client over-zoom

  • tile.minZoom / tile.maxZoom — the served zoom band.
  • Over-zoom past maxZoom is a client concern: set the map's GL-style source maxzoom to the layer's maxZoom. The renderer then keeps drawing the top-zoom tiles, scaled, instead of requesting tiles that don't exist.
i

tile.extent (e.g. 4096) is the MVT grid resolution within a single tile — unrelated to geographic extent or zoom. Leave it at 4096 unless you have a specific reason.

Frontend integration (OpenLayers)

Using OpenLayers + ol-mapbox-style:

// 1) Read layer metadata from YOUR endpoint.
const meta = await (await fetch('/vector-tile-hub/layers/82')).json();

// 2) Load a Mapbox GL style whose vector source points at YOUR tile route.
const style = await (await fetch('/ol-style.json')).json();

// 3) Clamp the source zoom range so the renderer OVER-ZOOMS beyond maxZoom
//    instead of requesting missing tiles.
const src = style.sources.vth;
if (Number.isFinite(meta.minZoom)) src.minzoom = meta.minZoom;
if (Number.isFinite(meta.maxZoom)) src.maxzoom = meta.maxZoom;

await olms.apply(map, style);   // ol-mapbox-style

The GL source's tiles URL is your tile route, e.g. "/vector-tile-hub/tiles/82/{z}/{x}/{y}.pbf". The Sample ships a working app.js, index.html, and an SLD→GL-style converter.

Recommended generation strategy

  1. Set layer.extent to the real data bounds (fast R-tree path + gating).
  2. Cap deep zooms with tile.maxGenerationZoom; let the deepest zooms generate on demand.
  3. Pre-generate with a sensible maxDegreeOfParallelism; watch progress in the dashboard.
  4. Use EnqueueNotifyChange / InvalidateAsync for incremental updates; EnqueueSwap for full blue/green rebuilds.
